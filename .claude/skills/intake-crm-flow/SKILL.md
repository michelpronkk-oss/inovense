# Skill: intake-crm-flow

Use this skill when:
- Building intake forms
- Designing lead capture flows
- Structuring CRM states
- Implementing autoresponder-ready backend logic

Objectives:
- Create a guided intake flow
- Capture service lane, budget, urgency, and context
- Store leads cleanly
- Support internal qualification and follow-up
- Keep v1 simple and scalable

Suggested lead statuses:
- new
- reviewing
- qualified
- proposal_sent
- call_booked
- won
- lost

Checklist:
1. Keep form short but useful
2. Filter for quality without hurting conversion
3. Validate all input
4. Make data model extensible
5. Support future automation
